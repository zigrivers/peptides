# Task 6.4 Plan — Vial Expiry Background Job

**Story:** PRD §5.2.6 (vial expiry)
**ADRs:** ADR-012 (Railway Cron)
**Reuses:** the `markOrdersStale` pattern from Task 3.4 (system-level
cron, defense-in-depth scoping, `actorUserId: 'SYSTEM'` audit).

The stale-orders cron already exists from Task 3.4; this task adds the
sibling vial-expiry cron.

---

## What ships

1. **Service** — `lib/reconstitution/application/VialExpiryService.ts`:
   - `markVialsExpired(now: Date)`: finds vials where
     `status === 'RECONSTITUTED'` AND `expiresAt < now`, transitions
     each one to `status: 'EXPIRED'`, emits a `VIAL_EXPIRED` audit per
     transition. Returns `{ expired: number }`. Idempotent.
2. **Cron route** — `app/api/cron/vial-expiry/route.ts`:
   - `POST` with `Authorization: Bearer ${CRON_SECRET}`.
   - Returns `{ expired }` on success, 500 on service error.
3. **Audit action** — `VIAL_EXPIRED` (category `Reconstitution`).
4. **AGENTS.md / CLAUDE.md exception** for the global findMany.

## Why a stored status (and not just badge-time computation)?

Vial badges are computed at read time (`VialService.toViewModel`), so
"EXPIRED" rendering already works without a cron. But three things
benefit from a persisted status flip:

1. **Audit history**: the user can see exactly when a vial flipped to
   expired in their audit log.
2. **Inventory count correctness**: `countActiveVialsForCompound` uses
   `status === 'RECONSTITUTED'` — once the cron transitions, the count
   correctly excludes expired vials with no extra read-time filter.
3. **Defense-in-depth for dose logging**: pre-checks that look at
   `vial.status` can refuse to log against an EXPIRED vial without
   needing to re-derive expiry from `expiresAt`.

## Out-of-scope

- Recovering an EXPIRED vial (intentional — the status transition is
  one-way; user can mark a new vial as RECONSTITUTED from a fresh dry
  vial if needed).
- Email/push notification on vial expiry — the daily reminder cron
  (Task 5.2) already reaches the user; surfacing expiry through that
  channel can be a follow-up.
