# Task 5.3 Plan — Subjective Outcomes + Correlation Timeline

**Stories:** US-TRK-06 (outcome logging), US-TRK-07 (correlation chart)
**Schema:** existing — `OutcomeLog` (one per user per day, unique constraint) and
`ProtocolRating` (1-5 per protocol per outcome). No migration needed.

---

## What ships

1. **Infrastructure** — `lib/tracker/infrastructure/OutcomeLogRepo.ts`:
   - `findForDate(userId, scheduledDate)` — single outcome for a calendar day.
   - `listInRange(userId, fromUTCMidnight, toUTCMidnight)` — for the chart.
   - `topTagsLastNDays(userId, n)` — frequency-ranked tags from the past N days.
   - `upsertWithRatings(...)` — atomic upsert of outcome + replace protocol ratings.

2. **Application** — extend `lib/tracker/application/OutcomeLogService.ts`:
   - `upsertOutcome(userId, input)` — Zod-validated; ProtocolRating rows
     replaced in the same transaction; userId-scoped throughout. Audit
     emits `OUTCOME_LOGGED` (first time) or `OUTCOME_UPDATED` (rev). Each
     ProtocolRating change emits `PROTOCOL_RATED` (so protocol-level rating
     history survives outcome-log churn) — capped to one audit per
     protocol per outcome upsert.
   - `getOutcomeForDate(userId, date)` — returns the log + nested ratings.
   - `getTimelineSeries(userId, days)` — returns daily buckets of
     `{ scheduledDate, doseEvents: number, outcomeRating: number | null }`
     for the chart. `doseEvents` counts logged doses (status=LOGGED) on
     that calendar day.
   - `getCorrelationStats(userId, days)` — returns average outcome on
     dosed vs not-dosed days within the window.
   - `getTopTagSuggestions(userId)` — 3 most-frequent tags in past 14 days
     (US-TRK-06 AC-3).

3. **Server actions** — `app/actions/tracker/`:
   - `log-outcome.ts` — single action handles both create and update via
     unique `(userId, scheduledDate)`.

4. **UI**:
   - `app/(dashboard)/tracker/outcomes/page.tsx` — RSC; loads today's outcome
     + last 30 days. Renders the form (today) and the SVG correlation chart.
   - `_components/OutcomeForm.tsx` (client) — rating buttons (1-5), tag
     chips (with frequent presets shown first), note textarea (1000-char
     limit), per-protocol rating sliders for the day's active protocols.
   - `_components/CorrelationTimeline.tsx` (server) — SVG dual-axis chart
     plus accessibility data-table (US-ANL-01 AC-7). 30/90-day toggle.
   - `_components/CorrelationStats.tsx` — average rating on dosed vs not-dosed.

5. **Audit actions** — add to `AuditEvent.ts` `Protocol` category:
   - `OUTCOME_LOGGED`
   - `OUTCOME_UPDATED`
   - `PROTOCOL_RATED`

## Identity scoping

- All Repo methods take `userId` as a required parameter and include it
  in every WHERE clause. No global scans.
- ProtocolRating ownership is verified inside `upsertWithRatings`: the
  set of protocolIds the client submits must be a subset of the actor's
  current protocols (`protocol.findMany WHERE userId AND id IN (...)`).
  Any extra id throws `protocol_not_owned`.

## Chart implementation note

- Pure SVG (no chart library). Width 100%; bars + line drawn from the
  bucket array. Data table is sibling content with `sr-only` styling on
  the chart container for screen readers (US-ANL-01 AC-7).
