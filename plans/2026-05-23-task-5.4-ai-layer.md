# Task 5.4 Plan — AI Layer (Vercel AI SDK + Anthropic + Gemini)

**ADR:** ADR-010 (AI Strategy)
**Story:** No direct user story for v1 — this is infrastructure that enables
Phase 3 features and admin-only profile drafting.

---

## What ships

1. **Module** — `lib/ai/`
   - `lib/ai/domain/types.ts` — provider + model literals; structured
     output shapes for the two v1 operations.
   - `lib/ai/domain/schemas.ts` — Zod schemas for the structured outputs.
   - `lib/ai/infrastructure/anthropicClient.ts` — lazy Proxy wrapper
     mirroring `lib/shared/email.ts` (Resend lesson). Module import is
     side-effect free; instantiation deferred until first use.
   - `lib/ai/infrastructure/geminiClient.ts` — same pattern for Gemini.
   - `lib/ai/application/AIClient.ts` — provider fail-over orchestrator:
     - Timeout (default 30s).
     - Retry-once with 1s backoff on transient errors (5xx, network).
     - Fall through Anthropic → Gemini.
     - Throws `ai_unavailable` when both providers fail.

2. **Use cases**
   - `lib/ai/application/extractCitation.ts` — given paper text, returns
     `{ title, authors[], journal, year, doi?, pmid? }` (Zod-validated).
     Uses `claude-haiku-4-5-20251001` per ADR-010.
   - `lib/ai/application/draftCompoundProfile.ts` — given a prompt +
     citations, returns a draft profile text. Uses `claude-sonnet-4-6`.
     Returns plain text; never auto-publishes. The draft is for human
     review at the admin compound profile editor.

3. **Server action** (admin-only) — `app/actions/admin/draft-profile.ts`
   - Power-User role gate, then call into `draftCompoundProfile`.
   - Returns the draft text to the admin UI (no DB write yet — the
     draft is reviewed and edited before publish in the existing profile
     editor; that wiring is left for a follow-up task).
   - Never throws to the user; AI failure returns an `error` field so
     the admin can fall back to manual writing.

4. **Eval tests** (`tests/evals/`)
   - `ai-citation-extraction.test.ts` — pinned input → expected output
     shape; uses a stubbed model (no real API calls in CI).
   - `ai-profile-draft.test.ts` — output is non-empty, contains
     expected sections, does not contain disallowed phrases ("safety
     clearance", "approved").

5. **Acceptance tests** (`tests/acceptance/`)
   - `AI-client.test.ts` — provider fail-over, timeout, retry behavior;
     verifies AI failures DO NOT block the caller.

6. **Audit events**
   - `AI_REQUEST_INITIATED` and `AI_REQUEST_FAILED` (`Security` category).
   - Metadata: provider, model, operation, latency-ms. **No prompt or
     response content** — operations §7 forbids logging prompts.

7. **AGENTS.md / CLAUDE.md**
   - Note: AI operations may run from a server context without a user
     session (PubMed cron in Phase 3). Until then, every call site is
     either an admin-only server action OR a cron route — both of which
     have their own auth. No new identity-scoping exception is needed.

## TDD: acceptance tests

`tests/acceptance/AI-client.test.ts`:

- AC-1: Both Anthropic + Gemini keys unset → throws `ai_unavailable`.
- AC-2: Anthropic succeeds → returns result; Gemini never called.
- AC-3: Anthropic fails with 5xx → retries once; succeeds → returns.
- AC-4: Anthropic fails twice → Gemini tried; succeeds → returns.
- AC-5: Both providers fail → throws `ai_unavailable`; emits
  `AI_REQUEST_FAILED` audit.
- AC-6: Anthropic returns malformed output (fails Zod) → falls through
  to Gemini.
- AC-7: Operation timeout (test with vi.useFakeTimers) → counted as
  failure; retry path engaged.

`tests/acceptance/AI-extractCitation.test.ts`:

- AC-1: returns valid CitationOutput shape on success.
- AC-2: invalid model output → throws `ai_invalid_response`.
- AC-3: keys missing → throws `ai_unavailable` (caller must catch).
